

Руководство по T-FLEX CAD Open API

# Введение  
    
Данное руководство описывает структуру, классы и методы программного интерфейса (API) системы автоматизированного проектирования T-FLEX CAD. Open API T-FLEX CAD построено на основе использования технологии .NET, которая обеспечивает ряд существенных преимуществ по сравнению с традиционными технологиями. К ним относятся:

  * Высокая производительность приложений
  * Простота расширяемости состава классов и функций системы, поддерживающей данную технологию
  * Большое количество языков программирования, поддерживающих данную технологию
  * Возможность использования технологий ActiveX и COM
  * Простота организации Open API, в полной мере использующего методы объектно-ориентированного программирования



Open API T-FLEX CAD предназначен для разработки приложений для системы T-FLEX CAD, использующих её функциональность. Приложения могут представлять собой отдельные модули (DLL), разрабатываемые во внешней системе программирования, а также небольшие программы (макросы), которые можно разрабатывать во встроенной в T-FLEX CAD среде программирования (редакторе макросов). Так же возможно создавать EXE-приложения – приложения, не связанные с системой, но использующие её функции.

**Система** \- T-FLEX CAD как приложение Windows. 

**Приложение** \- прикладной модуль, разработанный с использованием Open API, описываемого в данном руководстве. 

**Документ** \- единая совокупность данных (объектов документа или объектов модели), хранящихся в одном файле. 

**Вид документа** \- окно, в котором отображается документ. 

**Объект модели (объект документа)** \- элемент данных модели (геометрические компоненты чертежа или модели, переменные, спецификации и пр.) 

**Объект** \- элементарный экземпляр конкретного класса. Объект может быть как самостоятельным, так и зависимым от других объектов. 

**Команда** \- модальный процесс, реализующий взаимодействие с пользователем при помощи графического пользовательского интерфейса. Команда может использовать для работы вид документа, так и быть основана на диалоговом окне или последовательности диалоговых окон без использования вида документа. 

Все классы и структуры Open API T-FLEX CAD организованы в соответствии с их назначением в пространства имён (Namespaces). 

| Пространство имен | Назначение |
| --- | --- |
| [TFlex](N_TFlex.md) | Все классы и структуры Open API T-FLEX CAD |
| [TFlex.Command](N_TFlex_Command.md) | Реализация функциональности команд приложения, создание пиктографического меню, показывающего опции текущей команды |
| [TFlex.Drawing](N_TFlex_Drawing.md) | Прорисовка в окнах документах |
| [TFlex.Model](N_TFlex_Model.md) | Модель (документ) T-FLEX CAD, управление документом и всеми его элементами |
| [TFlex.Model.Model2D](N_TFlex_Model_Model2D.md) | 2D часть документа |
| [TFlex.Model.Model3D](N_TFlex_Model_Model3D.md) | 3D часть документа |
| [TFlex.Model.Model3D.Geometry](N_TFlex_Model_Model3D_Geometry.md) | Хранение геометрических данных в объектах 3D модели и обмена между ними ссылками на геометрию |
| [TFlex.Model.Model3D.Visual](N_TFlex_Model_Model3D_Visual.md) | Визуализация 3D модели |
| [TFlex.Dialogs](N_TFlex_Dialogs.md) | Элементы интерфейса пользователя для диалогов команд |
  
T-FLEX CAD поставляется в двух базовых вариантах - 2D (двухмерное проектирование и черчение) и 3D (трёхмерное моделирование). В 3D версию входит вся функциональность 2D версии системы. В связи с этим многие из типов объектов модели, относящихся к трёхмерному моделированию, можно создать или получить только в трёхмерной версии системы. Классы, объекты которых можно создать только в трёхмерной версии системы, реализованы в пространстве имён [TFlex.Model.Model3D](N_TFlex_Model_Model3D.md).
